# Component Guidelines

## 1. 기본 원칙

- 모든 공통 UI 컴포넌트는 props 기반으로 설계한다.
- 스타일과 로직을 분리한다.
- 컴포넌트는 특정 페이지에 의존하지 않는다.
  <br /><br />

## 2. 네이밍 규칙

- 공통 UI: `BaseBadge`
- 역할이 명확해야 하며, 추상적인 이름 사용 금지
  <br /><br />

## 3. 상태 표현 기준

- 사용자 인터랙션이 없는 경우 → BaseBadge
- 사용자 선택/조작이 필요한 경우 → Chip
  <br /><br />

## 4. 스타일 규칙

- size는 단계별로 정의
  - sm / md / lg로만 일단 정의하고 추후 추가가 필요할 경우만 더 늘릴 것
- color는 직접 값 전달 가능하지만, 디자인 토큰 사용 권장
